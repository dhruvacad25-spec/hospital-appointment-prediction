About This Project

A bootcamp project built like a production tool.

Logic Loop-Hospital No-Show Prediction started as a question: could a simple classifier, trained on ordinary appointment data, flag the visits a hospital is most likely to lose? This page walks through how it was built.

Tech stack
Python Flask Bootstrap 5 Scikit-learn Pandas NumPy Chart.js
Dataset

10,000 records

Historical hospital appointment data covering clinical, demographic, and behavioral fields.

25 features

From age and BMI to booking lead time and prior missed visits.

1 target

appointment_status — whether the patient attended or missed the visit.

Preprocessing pipeline
01

Dropped identifiers

The patient_id column was removed since it carries no predictive signal.

02

Filled missing values

Missing entries in chronic_disease were filled with "None" rather than dropped.

03

Encoded categorical fields

Label encoding was applied across all categorical columns to prepare them for the classifier.

04

Split & compared models

The data was split into train/test sets, and KNN was compared against a Decision Tree classifier.

Why KNN was selected

Across the same train/test split, K-Nearest Neighbors outperformed a Decision Tree classifier by over 12 points, making it the final model behind every prediction on this site.
